# 六十四卦白话解读 · 研究方案

## 一、问题分析

当前天问的解卦卡片显示的是：
1. 卦名 + 卦符
2. 文言卦辞（如"乾：元亨利贞"）
3. 爻辞列表
4. 变卦解读（如适用）

**痛点**：文言文门槛太高，普通用户看完卦辞后仍然不知道"这对我意味着什么"。

## 二、白话方案设计

### 每个卦需要增加的白话内容（~80 字）

结构：**一句话总结 + 生活场景映射**

示例（乾为天）：
> 💬 **白话**：这是最好的卦象之一。象征创造力全开、领导力爆棚的阶段。事业上大胆往前冲，感情里主动出击，投资可以积极布局。但要记住盛极必衰，锋芒太露容易惹人忌恨。

示例（坤为地）：
> 💬 **白话**：这是一个要"柔"不要"刚"的阶段。像大地一样包容、承载，不要强出头。事业上配合团队走、感情里多倾听少表态、投资保守观望。厚德载物才有好结果。

### 写法规则

1. 第一句给卦象定性（上上卦/中平卦/下下卦，或不直接标签而用语气暗示）
2. 中间 2-3 句：事业/感情/决策 的具体建议
3. 最后一句：注意事项或转折

### 每个卦控制在 60-120 字之间

## 三、数据结构设计

在 `gua-data.js` 的 `GUA_DATA` 中，每个卦增加一个 `plain` 字段：

```javascript
{
  name: "乾为天",
  binary: "111111",
  symbol: "䷀",
  judgement: "元亨利贞。",
  plain: "最好的卦象之一。象征创造力与领导力全开，...
  // ... 其余不变
}
```

## 四、渲染方式

在解读卡片中，卦辞下新增"💬 白话解读"区块，使用略有不同的背景色区分于文言卦辞。

## 五、工作量

- 64 卦 × ~100 字 = ~6400 字文案
- 在现有 `gua-data.js` 中追加字段即可
- 纯数据工作，不改逻辑

## 六、特殊处理

- 变卦解读也需白话：当出现变卦时，变卦的 `plain` 字段同步展示
- 梅花易数的体用生克结果也加一句白话总结
