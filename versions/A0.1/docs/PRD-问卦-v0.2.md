## 1. Executive Summary

**问卦 v0.2** 是一个纯前端、零依赖的 Web 端算卦应用。用户打开浏览器即可完成一次完整的铜钱掷爻起卦，获得六十四卦的卦辞爻辞解读。本版本交付核心掷爻→查表→解读的闭环体验。

---

## 2. Problem Statement

### 用户痛点

- **知识门槛高**：普通人想算卦，但不懂六十四卦、不知道爻辞含义、不会变卦解读
- **工具分散**：传统方式需要硬币+查书+手动对照，流程繁琐
- **解读不完整**：网上很多算卦工具只给卦名和一句笼统的话，缺少逐爻分析和变卦推演

### 业务影响

- 当前 `gua-data.js` 仅含 2/64 卦数据，`app.js` 掷爻逻辑全部为 TODO，产品不可用
- 缺少完整数据闭环导致所有下游功能（变卦、动画、梅花易数）无法推进

## 4. User Personas

### Persona A：好奇探索者 — 小陈

- **年龄**: 22-30 岁
- **场景**: 对传统文化感兴趣，周末闲暇想"算一卦玩玩"
- **行为**: 打开网页 → 点按钮 → 看结果 → 截图发朋友圈
- **需求**: 界面好看、操作简单、解读通俗易懂
- **痛点**: 看不懂文言文爻辞

### Persona B：易经爱好者 — 老周

- **年龄**: 35-50 岁
- **场景**: 有易经基础，遇事不决时正经起一卦
- **行为**: 认真凝神 → 掷爻 → 对照卦辞爻辞思考
- **需求**: 数据准确、有朱熹变占法、能看变卦
- **痛点**: 手工掷硬币+查书太慢，想要数字化辅助

## 5. User Stories

### US-1: 铜钱掷爻起卦（P0）

```
Feature: 铜钱掷爻起卦

  Scenario: 用户成功掷爻获得完整卦象
    Given 用户在主页
    When 点击"掷爻起卦"按钮
    Then 系统模拟六次铜钱掷爻
    And 每次掷爻展示该爻的结果（少阳/少阴/老阳/老阴）
    And 六次完成后展示本卦卦名、卦符、卦辞
    And 如有动爻，展示变卦卦名、卦符、卦辞

  Scenario: 掷爻过程中展示逐爻动画
    Given 用户点击了"掷爻起卦"
    When 每次掷爻完成
    Then 该爻从下往上逐步绘制到卦象展示区
    And 动爻（老阳/老阴）以特殊样式高亮标记

  Acceptance Criteria:
  - [ ] 点击按钮后，6 次掷爻按顺序执行
  - [ ] 每次掷爻的正面数(0-3)随机分布符合三枚铜钱概率模型
  - [ ] 本卦通过6位二进制码查 GUA_DATA 正确返回
  - [ ] 动爻位置正确标记，变卦正确计算
  - [ ] 无动爻时不展示变卦区域
```

### US-2: 卦象结果展示（P0）

```
Feature: 卦象结果展示

  Scenario: 展示本卦完整信息
    Given 掷爻已完成
    When 结果渲染完成
    Then 展示: 卦名（如"乾为天"）、卦符（䷀）、卦辞（元亨利贞）
    And 六爻爻辞逐条展示，初爻在最下，上爻在最上
    And 动爻以高亮样式区分

  Scenario: 朱熹变占法自动解读
    Given 掷爻已完成且存在 N 个动爻
    When 结果渲染完成
    Then 根据朱熹变占法规则自动选择解读焦点:
      | 0动爻 → 本卦卦辞
      | 1动爻 → 该动爻爻辞
      | 2动爻 → 两爻爻辞（以上爻为主）
      | 3动爻 → 本卦卦辞 + 变卦卦辞
      | 4动爻 → 变卦不变爻爻辞（以下爻为主）
      | 5动爻 → 变卦唯一不变爻爻辞
      | 6动爻 → 变卦卦辞（乾用九/坤用六）

  Acceptance Criteria:
  - [ ] 结果区展示卦名、卦符、卦辞
  - [ ] 六爻逐条展示且顺序正确（初爻在下）
  - [ ] 动爻高亮
  - [ ] 朱熹变占法焦点解读正确展示
  - [ ] 满6爻全动时乾显示"用九"、坤显示"用六"
```

## 6. Functional Requirements

### 6.1 数据层

| ID | 需求 | 优先级 |
|----|------|--------|
| FR-D01 | `GUA_DATA` 包含全部 64 卦，索引按通行本卦序 (1-64) | Must |
| FR-D02 | 每卦含 `id`, `name`, `symbol`, `binary`, `judgement`, `lines[6]` | Must |
| FR-D03 | 每卦含 `upperTrigram` 和 `lowerTrigram` 八卦符号 | Should |
| FR-D04 | 乾卦含 `yong: "用九：见群龙无首，吉。"` | Must |
| FR-D05 | 坤卦含 `yong: "用六：利永贞。"` | Must |

### 6.2 掷爻逻辑

| ID | 需求 | 优先级 |
|----|------|--------|
| FR-L01 | 模拟三枚铜钱：`Math.random()` × 3，0=反面，1=正面 | Must |
| FR-L02 | 正面数判定：2→少阳⚊, 1→少阴⚋, 3→老阴⚋→⚊, 0→老阳⚊→⚋ | Must |
| FR-L03 | 循环 6 次生成六爻数组，初爻 index=0，上爻 index=5 | Must |
| FR-L04 | 六爻编码为 6 位二进制：阳=1, 阴=0, 初爻=LSB(bit0) | Must |
| FR-L05 | 通过 binary 字段查 GUA_DATA 获取本卦 | Must |
| FR-L06 | 动爻位置处的 bit 翻转变换后再次查表获取变卦 | Must |
| FR-L07 | 无动爻时 bianGua = null | Must |

### 6.3 界面交互

| ID | 需求 | 优先级 |
|----|------|--------|
| FR-U01 | 掷爻过程逐爻动画展示（从下往上绘制） | Should |
| FR-U02 | 动爻以特殊样式高亮（如金色边框或闪烁） | Should |
| FR-U03 | 结果区展示本卦 + 变卦（如有）完整信息 | Must |
| FR-U04 | 六爻爻辞纵向排列，从初爻到上爻（自下而上） | Must |
| FR-U05 | 掷爻进行中禁用按钮防止重复点击 | Must |
| FR-U06 | 再次点击"掷爻起卦"清除上一次结果重新开始 | Must |

## 7. Non-Functional Requirements

| 类别 | 要求 |
|------|------|
| **性能** | 掷爻 + 查表 + 渲染 < 100ms（不含动画） |
| **兼容性** | Chrome/Firefox/Safari/Edge 最近 2 个主版本 |
| **Unicode** | 卦符 U+4DC0~U+4DFF 和八卦符号 U+2630~U+2637 正确渲染 |
| **离线** | 纯静态页面，不依赖任何 CDN 或后端 API |
| **可访问性** | 按钮有足够点击区域（≥44×44px），色相对比度 ≥ 4.5:1 |
| **代码规范** | 使用 ES5 兼容语法（`const`/`let`/箭头函数可接受），不引入构建工具 |

## 8. Technical Considerations

### 8.1 架构

```
index.html → app.js (主逻辑)
           → gua-data.js (纯数据，无逻辑)
           → animation.js (动画控制)
           → style.css (暗金主题)
```

### 8.2 数据模型（复用 research/00-研究总报告 定义）

```javascript
// Gua 对象
{
    id: 1,                    // 通行本卦序 1-64
    name: "乾为天",            // 完整卦名
    symbol: "䷀",              // Unicode 卦符
    binary: "111111",         // 6位二进制 (初爻=LSB)
    upperTrigram: "☰",       // 上卦符号
    lowerTrigram: "☰",       // 下卦符号
    judgement: "元亨利贞。",   // 卦辞
    lines: [                  // 6条爻辞，index 0=初爻
        "初九：潜龙勿用。",
        ...
    ],
    yong: null               // 用九/用六（仅乾坤）
}
```

### 8.3 二进制编码验证

关键检查点（research 报告指出的风险）：
- 咸卦(泽山咸) = 兑上艮下 → 110001 (49)
- 旅卦(火山旅) = 离上艮下 → 101001 (41)
- 损卦(山泽损) = 艮上兑下 → 001110 (14)

以上三卦 binary 值互不重复，原报告疑虑已排除。

### 8.4 无外部依赖

- 不使用 npm/webpack/vite
- 不使用任何第三方 JS 库
- 数据全部硬编码于 `gua-data.js`

## 10. Out of Scope

以下内容**明确不在 v0.2 范围内**：

- ❌ 梅花易数模式（三数起卦、互卦、体用生克）→ 移至 v0.3
- ❌ 模式切换标签 UI → 移至 v0.3（v0.2 仅铜钱模式）
- ❌ 铜钱 3D 抛掷动画 → 移至 v0.5
- ❌ 音效 → 移至 v0.5
- ❌ 卦象分享/导出功能 → 移至 v0.5
- ❌ 卦序歌诀学习模式 → 移至 v0.5
- ❌ 彖传、象传等扩展经文 → 体积控制，暂不加入
- ❌ 多语言支持 → 远期
- ❌ 后端 API / 数据库 → 永远不做（纯前端定位）
