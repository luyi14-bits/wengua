## 1. Executive Summary

**问卦 v0.3** 在铜钱掷爻之外新增**梅花易数起卦模式**。用户无需等待六次掷爻动画，只需输入三个数字（或一键随机/取当前时间）即可立即获得本卦、互卦、变卦三重卦象，并以体用生克五行关系判断吉凶。产品从单模式升级为双模式算卦工具。

---

## 2. Problem Statement

### 用户痛点

- **铜钱模式太慢** — 六次掷爻 + 动画需约 3 秒，只想快速算一卦的用户等不及
- **缺少深度解读** — 铜钱模式只看卦辞爻辞，用户想要更多维度的分析（互卦、体用生克）
- **数字起卦更贴近日常** — 很多人习惯报数字、看时间、翻页码起卦，这是梅花易数的天然优势
- **两种模式互补** — 铜钱解法重经文释义，梅花解法重五行生克，合在一起才是完整的算卦体验

### 业务影响

- 当前仅铜钱单一模式，产品功能单一，缺少差异化
- 梅花易数是 research 阶段就规划好的核心功能（见研究总报告 Phase 1/2），v0.2 完成铜钱闭环后自然推进

## 5. User Stories

### US-1: 模式切换（P0）

```
Feature: 双模式切换

  Scenario: 用户从铜钱模式切换到梅花易数模式
    Given 用户在铜钱掷爻模式下
    When 点击"梅花易数"标签
    Then 界面切换为梅花易数模式
    And 卦象展示区清空
    And 操作区变为三数输入界面
    And URL hash 变为 #meihua（可分享直达）

  Scenario: 模式切换不丢失另一模式的结果
    Given 用户在铜钱模式下已掷出一卦
    When 切换到梅花易数模式再切回铜钱模式
    Then 铜钱模式恢复上次结果（保留状态）

  Acceptance Criteria:
  - [ ] 顶部显示两个标签：[铜钱掷爻] [梅花易数]，当前激活标签高亮
  - [ ] 点击标签切换模式，切换动画流畅（≤300ms）
  - [ ] 每个模式独立管理自己的起卦结果
  - [ ] 标签区域在 480px 以下不换行
```

### US-2: 梅花三数起卦（P0）

```
Feature: 三数起卦核心算法

  Scenario: 用户输入三个数字起卦
    Given 用户在梅花易数模式下
    When 输入 15, 9, 20 并点击"起卦"
    Then 上卦数 = 15 % 8 = 7 → 艮☶
    And 下卦数 = 9 % 8 = 1 → 乾☰
    And 动爻 = 20 % 6 = 2 → 二爻动
    And 本卦为山天大畜䷙
    And 展示本卦+互卦+变卦+体用生克

  Scenario: 数字为0或负数的边界处理
    Given 用户输入负数
    When 点击"起卦"
    Then 提示"请输入正整数"
    And 不执行起卦

  Scenario: 一键随机数字
    Given 用户在梅花易数模式下
    When 点击"🎲 随机数字"
    Then 三个输入框自动填入 1-999 范围内的随机数
    And 自动执行起卦

  Scenario: 用当前时间起卦
    Given 用户在梅花易数模式下
    When 点击"🕐 用当前时间"
    Then 按年月日时法计算三个数
    And 自动填入并起卦

  Acceptance Criteria:
  - [ ] 上卦 = 第一个数 % 8 (余0→坤=8)
  - [ ] 下卦 = 第二个数 % 8 (余0→坤=8)
  - [ ] 动爻 = 第三个数 % 6 (余0→上爻=6)
  - [ ] 通过先天数(乾1兑2离3震4巽5坎6艮7坤8)映射到八卦
  - [ ] 上卦+下卦合成6位binary后查 GUA_BY_BINARY 得本卦
  - [ ] 输入验证：非空 + 正整数 + ≤9999
```

### US-3: 互卦计算（P0）

```
Feature: 互卦计算

  Scenario: 从本卦生成互卦
    Given 本卦六爻为 [上爻,五爻,四爻,三爻,二爻,初爻]
    When 计算互卦
    Then 取中间四爻 [五爻,四爻,三爻,二爻]
    And 上互卦(外互) = 五爻+四爻+三爻 → 生成上卦
    And 下互卦(内互) = 四爻+三爻+二爻 → 生成下卦
    And 上互卦+下互卦合成新卦 → 查 GUA_BY_BINARY 得互卦

  Scenario: 展示互卦在结果区
    Given 互卦已计算
    When 渲染梅花易数结果
    Then 互卦以独立卡片展示
    And 标注"互卦 — 揭示发展过程的中间状态"

  Acceptance Criteria:
  - [ ] 互卦取爻位置正确（5-4-3为外互，4-3-2为内互）
  - [ ] 4爻和3爻被上下互卦共用（重叠取爻法）
  - [ ] 任何本卦都有对应的互卦（共16种互卦）
```

### US-4: 体用生克五行判断（P0）

```
Feature: 体用生克五行吉凶

  Scenario: 体卦和用卦判定
    Given 动爻位置确定
    When 动爻在 1/2/3 爻（下卦区域）
    Then 体卦=上卦，用卦=下卦
    When 动爻在 4/5/6 爻（上卦区域）
    Then 体卦=下卦，用卦=上卦

  Scenario: 五行生克判断吉凶
    Given 体卦五行和用卦五行已知
    When 用卦五行生体卦五行
    Then 吉凶为"大吉 ✅"，关系为"用生体"
    When 体用同五行
    Then 吉凶为"吉 ✅"，关系为"体用比和"
    When 体卦五行克用卦五行
    Then 吉凶为"小吉 ➡️"，关系为"体克用"
    When 体卦五行生用卦五行
    Then 吉凶为"泄气 ⚠️"，关系为"体生用"
    When 用卦五行克体卦五行
    Then 吉凶为"凶 ❌"，关系为"用克体"

  Scenario: 结果区展示体用生克
    Given 体用生克已判定
    When 渲染梅花易数结果
    Then 展示体卦+用卦名称及五行
    And 展示生克关系+吉凶等级
    And 展示简短解读文字

  Acceptance Criteria:
  - [ ] 八卦五行映射: 乾兑金/离火/震巽木/坎水/艮坤土
  - [ ] 五行相生: 木→火→土→金→水→木
  - [ ] 五行相克: 木→土→水→火→金→木
  - [ ] 5种吉凶等级视觉区分（颜色+图标）
```

### US-5: 梅花结果渲染（P0）

```
Feature: 梅花易数结果区

  Scenario: 完整结果展示结构
    Given 梅花易数起卦完成
    When 渲染结果
    Then 结果区依次展示:
      - 本卦卡片（卦名+卦符+卦辞+数理来源）
      - 互卦卡片（卦名+卦符+取爻说明）
      - 变卦卡片（卦名+卦符+卦辞+动爻标注）
      - 体用生克卡片（体卦+用卦+五行+生克关系+吉凶+解读）

  Acceptance Criteria:
  - [ ] 四张卡片纵向排列，每张独立区分
  - [ ] 动爻在变卦中标注变化
  - [ ] 体用生克卡片用不同颜色背景区分吉凶等级
```

## 8. Technical Considerations

### 8.1 先天数 ↔ 二进制映射

梅花易数用**先天八卦数**（乾1兑2离3震4巽5坎6艮7坤8），与铜钱模式的**伏羲二进制序**（乾7兑6离5震4巽3坎2艮1坤0）不同。需要建立双向映射表：

```javascript
// 先天数 → 八卦符号
const XIANTIAN_TRIGRAM = [null, '☰','☱','☲','☳','☴','☵','☶','☷'];

// 八卦符号 → 3位二进制 (用于合成GUA_BY_BINARY查表key)
const TRIGRAM_TO_BINARY = {
    '☰': '111', '☱': '110', '☲': '101', '☳': '100',
    '☴': '011', '☵': '010', '☶': '001', '☷': '000'
};
```

### 8.2 互卦取爻法

```javascript
// benGua.binary = "111001" (初爻在右=LSB)
// 爻位: bit5=上爻, bit4=五爻, bit3=四爻, bit2=三爻, bit1=二爻, bit0=初爻
// 外互(上互)取 bit4+bit3+bit2 → 即 binary[1]+binary[2]+binary[3]
// 内互(下互)取 bit3+bit2+bit1 → 即 binary[2]+binary[3]+binary[4]
```

### 8.3 体用判定

```javascript
// 动爻位置: 1-6 (1=初爻...6=上爻)
// 动爻在1/2/3 → 下卦为动 → 体卦=上卦, 用卦=下卦
// 动爻在4/5/6 → 上卦为动 → 体卦=下卦, 用卦=上卦
```

### 8.4 复用 v0.2 资产

- `GUA_BY_BINARY` 查表 — 直接复用（铜钱和梅花都查同一个表）
- `getInterpretationFocus()` — 复用（动爻数=1的情况）
- `renderResult()` — 部分复用（卦名/卦符/爻辞渲染逻辑）
- CSS 暗金主题变量 — 完全复用
