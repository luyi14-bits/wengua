# 问卦项目 · 研究总报告

## 一、项目定位

**问卦 (WenGua)** — 一个精致的 Web 端算卦小玩具。

两种起卦模式：**铜钱掷爻**（传统六爻）+ **梅花易数**（数字起卦）。

目标：用户打开浏览器，即可获得完整的六十四卦解读体验。纯前端、零依赖、开箱即用。

## 二、核心算法路线

### 2.1 模式一：铜钱掷爻法

```
用户点击"掷爻起卦"
    ↓
模拟三枚铜钱 × 6次 (Math.random)
    ↓
每次判定: 0正面=老阳, 1正面=少阴, 2正面=少阳, 3正面=老阴
    ↓
生成六爻数组 [初爻, 二爻, 三爻, 四爻, 五爻, 上爻]
    ↓
编码为6位二进制 (阳=1, 阴=0, 初爻=LSB)
    ↓
查表 → 本卦卦名 + 卦符 + 卦辞
    ↓
动爻位置 → 翻转 → 变卦卦名 + 卦符 + 卦辞
    ↓
朱熹变占法 → 决定展示卦辞还是爻辞
    ↓
渲染: 逐爻动画 → 本卦 → 变卦(如有) → 解读文本
```

### 2.2 模式二：梅花易数法

```
用户输入三个数字 (或点击随机/取当前时间)
    ↓
第一个数 % 8 → 先天数查八卦 → 上卦
第二个数 % 8 → 先天数查八卦 → 下卦
    ↓
上卦 + 下卦 → 本卦
    ↓
第三个数 % 6 → 动爻位置 (1~6)
    ↓
动爻翻转 → 变卦
    ↓
本卦中间四爻 (2-5爻) 重叠取卦 → 互卦
    ↓
动爻所在 → 分出体卦/用卦
    ↓
体用五行生克 → 判吉凶层级
    ↓
渲染: 本卦 → 互卦 → 变卦 + 体用生克解读

## 三、数据模型

### 3.1 爻 (Yao)

```javascript
{
    position: 0~5,        // 0=初爻, 5=上爻
    type: 'yang'|'yin',   // 当前阴阳 (动爻按本卦)
    changing: boolean,     // 是否动爻(老阳/老阴)
    value: 0|1,           // 二进制值 (阳=1, 阴=0)
    label: string          // 初九/九二/.../上六
}
```

### 3.2 卦 (Gua)

```javascript
{
    id: 1~64,                       // 通行本卦序
    name: "乾为天",                  // 卦名
    alias: "乾",                     // 简称
    symbol: "䷀",                    // Unicode 卦符
    upperTrigram: "☰",              // 上卦符号
    lowerTrigram: "☰",              // 下卦符号
    binary: "111111",               // 6位二进制 (初爻LSB)
    judgement: "元亨利贞。",          // 卦辞
    lines: [                        // 六爻爻辞
        "初九：潜龙勿用。",
        "九二：见龙在田，利见大人。",
        ...
    ],
    yong: null|string               // 用九/用六（仅乾坤）
}
```

### 3.3 起卦结果 (Result)

```javascript
{
    benGua: Gua,              // 本卦
    bianGua: Gua|null,        // 变卦（无动爻时为null）
    yaoLines: Yao[],          // 六个爻
    changingIndices: number[], // 动爻位置 [0,1,...5]
    focus: {                  // 解读焦点（朱熹变占法）
        type: 'guaci'|'yaoci'|'yongjiu'|'yongliu',
        primary: string,
        secondary: string|null
    }
}
```

## 四、关键技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 起卦方法 | 铜钱掷爻法 + 梅花易数 | 双模式覆盖传统和数字起卦 |
| Unicode 卦符 | ✅ 使用 | U+4DC0~U+4DFF 原生显示，无需图片 |
| 八卦符号 | ☰☱☲☳☴☵☶☷ | U+2630~U+2637 原生支持 |
| 二进制编码 | 伏羲先天序 (初爻=LSB) | 可直接映射 0~63，编程友好 |
| 梅花先天数 | 乾1兑2离3震4巽5坎6艮7坤8 | 邵雍体系，与二进制序不同 |
| 解卦规则 | 朱熹变占法 + 体用生克 | 铜钱用朱熹，梅花用体用 |
| 卦辞爻辞 | 仅取经文（卦辞+爻辞） | 体积可控，彖传象传过于庞大 |
| 动画风格 | 暗金主题 + 铜钱翻转 | 符合算卦仪式感 |

## 五、已知问题与待验证

1. **伏羲二进制序 → 通行本周易序** 的映射表需要精确校对。网上的二进制序对应表有几处错误（如咸卦既出现在 001110 又可能与旅卦混淆）。

2. **爻辞文本** — 目前已有乾~复（24卦）的完整经文，剩余 40 卦的经文需要后续采集。

3. **Unicode 卦符渲染** — 需在所有主流浏览器上验证显示效果。部分旧系统可能缺少字体支持。
